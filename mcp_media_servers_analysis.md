# Анализ: Перенос медиа-инструментов на локальные MCP-серверы

## Цель
Отключить встроенные инструменты [Media](file:///c:/Users/Bochek/openclaw-project/temp_goclaw/internal/http/media_serve.go#15-18) в GoClaw (генерация и чтение аудио, видео, изображений) и заменить их на MCP (Model Context Protocol) серверы. Эти серверы будут запущены внутри Docker на локальной Windows-машине пользователя, которая объединена с сервером на Render через сеть Tailscale.

## Инструменты для замены (из скриншота)
1. **Create Audio** (Генерация аудио)
2. **Create Image** (Генерация изображений)
3. **Create Video** (Генерация видео)
4. **Read Audio** (Анализ аудио / Whisper)
5. **Read Image** (Анализ изображений / Vision)
6. **Read Video** (Анализ видео)

## Архитектурное решение
1. **Хостинг MCP-серверов:** Локальная Windows-машина с установленным Docker Desktop. Мы создадим [docker-compose.yml](file:///c:/Users/Bochek/openclaw-project/docker-compose.yml), который поднимет необходимые MCP-серверы.
2. **Связь (Tailscale):** Windows-машина имеет свой собственный IP в сети Tailscale (например, `100.x.y.z`).
3. **Транспорт MCP:** Так как Render-агент находится удаленно, мы будем использовать транспорт **SSE (Server-Sent Events) поверх HTTP**. MCP-серверы должны отдавать порты наружу или проксироваться на Windows-машине.
4. **Подключение в GoClaw:** В панели администратора GoClaw мы зайдем в раздел `MCP Servers` и добавим наши новые серверы, указав URL вида `http://[tailscale-ip-windows]:[port]/sse`.

## Варианты реализаций MCP-серверов
Для обработки медиа на Windows потребуется выбрать, откуда брать мощности:
* **Локальные модели:** Если на Windows есть мощная видеокарта (Nvidia RTX), можно поднять MCP-серверы, которые локально запускают Stable Diffusion (для картинок) и Whisper (для аудио) без внешних API.
* **API-обертки:** Если локальная видеокарта слабая, MCP-серверы могут просто служить мостом к бесплатным/платным внешним API (например, к открытым моделям HuggingFace, OpenAI и т.д.).

## Рекомендация по организации проекта (Ответ на вопрос)
Мы проделали огромный путь по деплою самого движка GoClaw на Render. Текущая папка `openclaw-project` логически посвящена именно **самому агенту и его деплою**.

**Настоятельно рекомендуется создать новую папку** (например, `C:\Users\Bochek\mcp-servers-windows`) и запустить новый инстанс Antigravity именно в ней. 

**Почему новый проект лучше?**
1. **Чистый контекст:** У ИИ будет абсолютно чистая память, сфокусированная только на написании Python/Node.js кода для MCP-серверов и Docker. История о Render, Tailscale-роутинге агента и багах GoClaw не будет отнимать токены и сбивать фокус.
2. **Независимый репозиторий:** MCP-серверы — это инфраструктура вашей Windows-машины. Их логично хранить в отдельном Git-репозитории.
3. **Разделение рисков:** Ошибки при конфигурации локального Docker не затронут рабочую конфигурацию [render.yaml](file:///c:/Users/Bochek/openclaw-project/render.yaml) и [Dockerfile](file:///c:/Users/Bochek/openclaw-project/Dockerfile) агента Эхо.
